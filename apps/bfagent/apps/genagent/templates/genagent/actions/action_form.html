Action Guide
What is an Action?

Actions are individual tasks executed within a Phase. Each action uses a Handler to perform specific operations like data validation, transformation, or external API calls.

Execution Order

Actions execute sequentially based on their order number. Lower numbers run first.

Handler Selection

Choose a handler that matches your task. After selection, configuration options specific to that handler will appear automatically.

Configuration

Each handler may require specific configuration. Fields are generated dynamically based on the handler's schema.

Error Handling

  • Retry: Number of retry attempts on failure
