### Secret-Optimierung (917c81d)

- `GHCR_TOKEN` entfernt → nutzt `secrets.GITHUB_TOKEN` (built-in, `packages: write`)
- `PROD_SSH_KEY` → `DEPLOY_SSH_KEY` (existiert bereits in risk-hub, bfagent, travel-beat, billing-hub)
- `PROD_HOST` → `DEPLOY_HOST` (existiert bereits)
- `PROD_USER` → `DEPLOY_USER` (existiert bereits)
- Repos mit bestehenden Deploy-Secrets funktionieren **sofort ohne manuelle Konfiguration**
- Staging-Secrets (`STAGING_SSH_KEY` etc.) bleiben optional

### Repos ohne bestehende Deploy-Secrets (manuelle Einrichtung nötig)

Repos die noch nie direkt deployt haben (z.B. coach-hub, wedding-hub, 137-hub, writing-hub,
research-hub, ausschreibungs-hub) brauchen `DEPLOY_SSH_KEY`, `DEPLOY_HOST`, `DEPLOY_USER`
als Repository Secrets in GitHub UI → Settings → Secrets and variables → Actions.
